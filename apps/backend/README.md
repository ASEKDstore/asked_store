# ASKED Store Backend

Backend приложение для ASKED Store на Node.js + Express + TypeScript + Prisma.

## Установка и запуск

```bash
cd apps/backend
npm install
npm run dev
```

## Переменные окружения

### Обязательные переменные

- `DATABASE_URL` - URL подключения к базе данных PostgreSQL
- `JWT_SECRET` - Секретный ключ для JWT токенов

### Telegram уведомления

Для работы уведомлений о заказах необходимо настроить следующие переменные:

- `TELEGRAM_BOT_TOKEN` - Токен Telegram бота для отправки уведомлений
- `TELEGRAM_ADMIN_IDS` - Список ID администраторов через запятую (например: `123456789,987654321`)

**Важно:** 
- Администраторы и клиенты должны нажать кнопку "Start" боту хотя бы один раз, иначе личные сообщения (ЛС) не будут доставляться
- Если пользователь не начал диалог с ботом, уведомления не будут отправлены, но это не повлияет на создание заказа

### Опциональные переменные

- `TELEGRAM_ADMIN_CHAT_ID` - ID чата для отправки уведомлений (альтернатива TELEGRAM_ADMIN_IDS)
- `PORT` - Порт сервера (по умолчанию: 3000)
- `NODE_ENV` - Окружение (development/production)

## API Endpoints

### Заказы

- `POST /api/orders` - Создание нового заказа
- `GET /api/orders?tgId=...` - Получение заказов по Telegram ID
- `PATCH /api/orders/:id` - Обновление статуса заказа

## Уведомления

При создании заказа автоматически отправляются уведомления:

1. **Администраторам** - в личные сообщения (если настроен `TELEGRAM_ADMIN_IDS`)
2. **Клиенту** - подтверждение заказа с номером и составом

Уведомления не блокируют создание заказа - если отправка не удалась, заказ всё равно будет создан.
